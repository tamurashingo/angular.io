5 Min Quickstart

  • Angular 2 for TypeScript
  • Angular 2 for JavaScript
  • Angular 2 for Dart
  • Angular 1 for JavaScript
  • Angular 1 for Dart

ごくシンプルなAngular2アプリケーションをTypeScriptでゼロから作ってみましょう。

TypeScriptは欲しくない？

ここではTypeScriptで始めますが、バナーのコンボボックスから言語を選択することで、 JavaScriptやDartでAngular2アプリを書くことができます。

もっとも短くて、もっとも早く ...

できるだけ早くAngular2で画面に何か表示してみましょう。

今まさに開発マシン上で行う手順について説明しようとしていますが、 同じ手順をplunkerのようなインタラクティブで、 オンラインな環境で行うことができます。 アプリを動かすためにサーバをインストールする必要はありません。

もし気に入った場合は、そして気に入るだろうと思いますが、後で実際のマシン上で繰り返して行うことができます。

アプリケーションプロジェクトのための新しいフォルダを作ります。 こんな感じです。

mkdir angular2-quickstart
cd    angular2-quickstart

最初のAngularコンポーネント

app.tsという新しいファイルを作成し、以下のコードをコピペします。

app.ts
BAD FILENAME: ../../_fragments/quickstart/ts/src/app/app.ts.md Current path: docs,ts,latest,quickstart PathToDocs: ../../

最も重要なAngular2の機能の一つであるAngular2コンポーネントを定義しました。 コンポーネントは アプリケーションビューを作って、アプリケーションロジックでサポートする、 基本的な手法です。

空の何もしないクラスAppComponentです。 実際のアプリケーション開発をする準備ができたら、 プロパティとアプリケーションロジックで膨らんでいくでしょう。

上記のクラスで、@Componentデコレータを確認できます。

メソッド名の前にある@は、Componentをデコレータとしてみなします。 "デコレータ"はTypeScriptの機能で、クラスのメタデータを作るためにあります。 AngularはコンパイルされたJavaScriptからこのメタデータを探し、適切に応答します。

@ComponentはこのクラスがAngularのコンポーネントであることをAngularに教えます。 @Componentメソッドに移動した設定オブジェクトは、2つのフィールド、 selectorとtemplateを持ちます。

selectorは、my-appと名づけられたHTML要素のためのCSSセレクタを定義します。 Angularはmy-app要素に出くわしたら、AppComponentインスタンスを作成し、表示します。

templateフィールドはコンポーネントの仲間のテンプレートで、 Angularへどのようにビューをレンダリングするのかを伝えます。 このテンプレートは "My First Angular App" と話す一行のHTMLです。

bootstrapメソッドは、アプリケーションルートでこのコンポーネントを使ってアプリケーションを開始することを、 Angularに伝えます。 そのうち、アプリケーションはこのルートからツリー状に発生するコンポーネントで構成されるであろうことが推測できるでしょう。

先頭行で、ComponentとbootstrapをAngular2ライブラリからインポートしました。 今はそういうものです。 もはやグローバルな名前空間から自分のコードやライブラリのコードを探し出さなくてよいのです。 ファイルやライブラリから必要なときに必要なものをimportします。

index.htmlの追加

index.htmlファイルの作成

下記のコードをコピペして ... 議論しましょう:

index.html
BAD FILENAME: ../../_fragments/quickstart/ts/src/index.1.html.md Current path: docs,ts,latest,quickstart PathToDocs: ../../

3つの特筆すべきHTMLのセクションを見ていきます:

  1. ウェブからJavaScriptライブラリを読み込みます。 議論なしでそのまま信用しましょう。

  2. Systemと呼ばれるもので設定を行い、 先ほど書いたAppComponentのアプリケーションファイルをインポートするよう依頼しています。 Systemはモジュールローダー(system.jsライブラリから読み込み)で、 コードをimportできるツールです; import文がAppComponentにあったのを覚えていますか？

    また、ブラウザのまさしくここでTypeScriptソースコードからJavaScriptへ"トランスパイル" (別名"コンパイル") するようsystem.jsに依頼しています。

  3. <body>の<my-app>タグに気付きます。 それはカスタムHTML要素であり、 先ほど定義した@Componentデコレータで装飾されたAppComponentクラスです。

実行！

ブラウザがアプリケーションを読み込むために、静的ファイルサーバが必要です。

手元にファイルサーバがない？ npmパッケージマネージャを使ってlive-serverと呼ばれるお気に入りの中の一つをインストールしましょう。

npmをがない？ 今使うのと、このドキュメントを通して繰り返し使うので、 入手してください。

npmのインストールが終わったら、ターミナルを開いて、以下のように打ち込んでください。

npm install -g live-server

ターミナルを開いて、以下のように打ち込んでください。

live-server

ちょっと待つと、ブラウザが立ち上がって表示されます。

おめでとうございます！営業中です。

代わりにLoading...が表示されている場合、 付録 ES6をサポートするブラウザを参照してください。

どうしました？

大急ぎで稼動させましたが、こんなふうにかなり長い間Angularを調査することができます。

いくつもの理由から、これはアプリケーションの構築には良い方法ではありません。

  • アプリのファイルが増えると、ブラウザ上でTypeScriptをコンパイルするのはうんざりするほど遅くなります。 もちろんプロダクション環境ではやらないでしょう。 ローカルでコンパイルし、生成されたJavaScriptをサーバに配置することを学ばなければなりません。 そのためにツールが必要になるでしょう。
  • ウェブからJavaScriptライブラリをダウンロードするのはデモには良いですが、開発を遅くします。 毎回アプリがリロードすると、これらのライブラリを再取得します。 ブラウザのキャッシュを当てにしてはいけません。 デバッグとライブリロードの技術でブラウザキャッシュは使えないでしょう。 また、ウェブからライブラリをロードするのはオフラインでの開発や接続が貧弱な環境での開発ができなくなります。 マシンにライブラリを落としてきて、ローカルで使うことを学びましょう。
  • 開発サイクルをできるだけ早く、できるだけ摩擦なしにしたいです。 コードを変更する際、ブラウザで結果をすぐに確認したいです。 このためのツールと手順があります。

一生懸命やる

開発をよりよい基礎にするため、いくつかのステップが必要です。

  1. 将来の成長のためアプリケーションプロジェクトの構造を見直す
  2. いくつかのツールとパッケージをインストールする
  3. ローカルのライブラリを使うようにindex.htmlを変更する
  4. TypeScriptのコンパイルをローカルで行う、また変更を見張る

ターミナルで動いているlive-serverを止め(Ctrl-C)、以下の手順を始めます。

将来の成長のためアプリケーションプロジェクトの構造を見直す

今のところ、全部を "angular2-quickstart" のルートフォルダに置いています。 ファイルが2つだったら悪くはありません。 アプリケーションが発展するには良くはありません。

プロジェクトに少し構造を与えましょう。

プロジェクトのソースコードを配置するため src というサブフォルダと、 アプリケーションのソースコードを配置するため src/app というサブサブフォルダを追加します。

OS/XとLinuxでは:

mkdir src/app

Windowsでは:

mkdir src\app

index.htmlをsrcフォルダに移動します。

app.tsをsrc/appフォルダに移動します。

プロジェクトのフォルダはこんな風に見えるはずです。

angular2-quickstart
└── src
    ├── app
    │   └── app.ts
    └── index.html

npmパッケージのローカルインストール

index.htmlのウェブベースのスクリプトをローカルマシンのスクリプトと置き換えていきます。

angular2 - Angular2ライブラリ

systemjs - モジュールローディングを提供するオープンソースライブラリ

また、二つの開発ツールもインストールします:

typescript - TypeScriptコンパイラ

live-server - ファイルの変更時にブラウザをリロードする静的ファイルサーバ 以前はロードできました。 もう一度プロジェクト内でローカルに置くことで、 グローバルサーバからのアンインストールやバージョンの変更があっても脆弱ではなくなります。

アプリケーションのルートフォルダでターミナルを開いてください。

これらのコマンドを入力:

npm init -y
npm i angular2@2.0.0-alpha.44 systemjs@0.19.2 --save --save-exact
npm i typescript live-server --save-dev

これらのパッケージインストールコマンドとnpmのpackage.json作成コマンドは、 アプリケーションの開発と将来のメンテナンスに役立ちます。 package.jsonの本質は次のようなものです:

ERROR: Unable to extract json using config: "[object Object]"

また、scriptセクションがあります。以下のように書き換えてみましょう。

ERROR: Unable to extract json using config: "[object Object]"

ちょうどプロジェクトをこれからすぐに実行するであろうスクリプトコマンドで拡張し終えました。

index.htmlの更新

ライブラリスクリプトセクションの参照をにインストールしたパッケージのスクリプトに置き換えましょう。

BAD FILENAME: ../../_fragments/quickstart/ts/src/index-libraries.html.md Current path: docs,ts,latest,quickstart PathToDocs: ../../

Systemスクリプトの設定を以下のように更新しましょう。

BAD FILENAME: ../../_fragments/quickstart/ts/src/index-systemjs.html.md Current path: docs,ts,latest,quickstart PathToDocs: ../../

ブラウザ上でTypeScriptのコンパイルはこれ以上行いません。 マシン上で行い、生成されたJavaScriptファイルをサーバに配置します。

JavaScriptファイルの拡張子がデフォルトで.jsとなるよう、system.jsを設定しなおさなくてはいけません。 いつか、Fooクラスをfoo.tsというファイルで追加し、このようにインポートするかもしれません。

import {Foo} from './app/foo'

system.jsはsrc/appフォルダでfoo.jsというファイルを探せばよいことを知っています。

最後の行で行ったのがこれになります。 メインのアプリケーションファイルapp(正確に言うと生成されたapp.js)をsrc/appフォルダ(移動させたのを覚えてますか？)からインポートしています。

これが最終版です

index.html
BAD FILENAME: ../../_fragments/quickstart/ts/src/index.html.md Current path: docs,ts,latest,quickstart PathToDocs: ../../

TypeScriptのコンパイルの準備

TypeScript設定ファイルの追加

エディタにTypeScriptコードの解析方法を伝えるため、 またこれからすぐに実行するであろうTypeScriptのコンパイルコマンドを単純化するために、 tsconfig.jsonという設定ファイルを追加します

srcフォルダに行き、以下の内容でtsconfig.jsonファイルを作ってください:

tsconfig.json
BAD FILENAME: ../../_fragments/quickstart/ts/src/tsconfig.json.md Current path: docs,ts,latest,quickstart PathToDocs: ../../

このファイルとこれらの設定についてより詳しい内容は付録 TypeScriptの設定を参照してください。

最終的な構成

プロジェクトフォルダの構成はこうなります:

angular2-quickstart
├── node_modules
├── src
│    ├── app
|    │    └── app.ts
│    ├── index.html
│    └── tsconfig.json
└── package.json

TypeScriptからJavaScriptへのコンパイル

もはやブラウザ上でTypeScriptからJavaScriptへのコンパイルは行いません。 マシン上でTypeScript Compiler (TSC)を実行します。

アプリケーションのルートフォルダでターミナルウィンドウを開いて、入力します:

npm run tsc

終わったらsrcフォルダに生成されたapp.jsファイルが見つかります。 またデバッガがJavaScriptとTypeScript間をナビゲートするのに役立つapp.map.jsファイルも見つかります。

スクリプトにコンパイラ監視オプション(-w)をセットしていたので、 コンパイルが終わったときにコンパイラが生き続けます。 コンパイラは.tsファイルの変化を監視し、自動的に再コンパイルします。

実行中のターミナルは放置しておきましょう。 Ctrl-Cでいつでも止められます。

アプリの実行！

今、作業中のアプリを見る準備ができました。

アプリケーションフォルダのルートで別のウィンドウを立上げlive-serverをもう一度起動しますが、 今回はnpmスクリプトコマンドでやっていきます:

npm start

live-serverはブラウザを立上げ、HTMLとJavaScriptを配信し、 もう一度アプリケーションメッセージを表示します。

変更する live-serverはファイルの変更を検出し、自動的にブラウザをリフレッシュします。

メッセージを"My SECOND Angular 2 app"に変更してみましょう。

最初のターミナルウィンドウのTypeScriptコンパイラがソースコードを監視しています。 これが再コンパイルを行い、app.jsを更新します。 live-serverはその変更を検出し、ブラウザをリロードさせます。

ターミナルのlive-serverは実行し続けます。Ctrl-Cで止めることができます。

何をしましたか？

最初のアプリケーションはあまり良くありません。 基本的にAngular2の"Hello, World"です。

最初はシンプルにしました: 小さいAngularのコンポーネントを書き、 index.htmlにJavaScriptライブラリを追加し、 静的ファイルサーバを使って起動しました。 これが"Hello, World"アプリに期待したすべてです。

もっと大きい野心があります。

Angularに"Hello, World"アプリを作るよう依頼しませんでした。 洗練された要求で洗練されたアプリケーションを作るのに役立つよう依頼しました。

そのため、より大きいゴールに近づくための戦略的な技術投資を得ることができました。

  • ローカルにインストールされたライブラリを使って高速に起動し、 希望すればオフライン環境でも開発ができます。

  • TypeScriptをプリコンパイルしています。

  • コンパイラやlive-serverをコマンドで実行しており、変更したらすぐにフィードバックを得ることができます。

セットアップのオーバーヘッドが(大部分は)後ろに隠れている、ということは良いニュースです。 今まさに、Angular2で構築できるすばらしい点をデモする小さいアプリケーションを構築しょうとしています。

付録: TypeScriptの設定

コンパイラにJavaScriptファイルを生成させるため、
TypeScript設定ファイル(tsconfig.js)をプロジェクトに追加しました。 公式のTypeScript wikiで、 tscnfig.jsについて詳細を知ることができます。

noImplicitAnyフラグについてちょっと議論したいと思います。 TypeScript開発者はtrueにすべきかfalseにすべきか意見があいませんでした。 正解はありませんし、フラグは後から変更できます。 しかし、大規模プロジェクトでは現時点の選択が違いを生み出すので、議論に値します。

noImplicitAnyフラグがfalseとしたとき、 変数の使われ方から型推論ができない場合は、 コンパイラは黙って型をanyにします。 これが"暗黙のany"という意味です。

noImplicitAnyフラグがtrueでTypeScriptコンパイラが型を推論できなかったとき、 それでもJavaScriptファイルは生成しますが、エラー報告も行います。

このプロジェクトやこの開発者ガイドのほかの例では、noImplicitAnyフラグをfalseにセットしています。 厳格な型チェックを好む開発者はnoImplicitAnyフラグをtrueにセットすべきでしょう。 それがベストな選択に見えても、私たちは依然として変数の型をanyにセットします。 問題について検討した後で、明確化するかもしれません。

もしnoImplicitAnyフラグをtrueにした場合、同様に暗黙的なインデックスエラーとなるでしょう。 もしそれらの情報で役立つよりイライラさせられたら、 このフラグをつけることで抑制することができます。

"suppressImplicitAnyIndexErrors":true

付録 ES6をサポートするブラウザ

Angular2は、最もモダンなブラウザで見られるような、ES6サポートが必要です。 古いブラウザ(IE11を含む)は、必要な機能を得るためにshim(クサビ)を使うことができます。

package.jsonを作ったあと(クイックガイドの中間くらい)、 このコマンドを実行してプロジェクトにshimを追加します。

npm install es6-shim --save

index.htmlで他のスクリプトの前でshimを読み込むことができます。

<script src="../node_modules/es6-shim/es6-shim.js"></script>
